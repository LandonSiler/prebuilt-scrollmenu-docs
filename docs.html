prebuilt-scroll-menus

Documentation

> Usage

Click the bot (acting as a button) titled: "Create".

A menu should then appear prompting you for a few Inputs which are described below.

If a menu does not appear, you may try reloading the inst, or if you're a developer, ensure no scripts are activly overwriting the menu. configBot.tags.menuPortal = "scrollMenuCreationInput";

Your Inst: ${url}

> User Input

Field: Unique Name.

This ultimately is the Dimension that the scroll menu is created / stored / acessed in.

This text field will not let you enter the name of any existing Dimension.

It is best practice to make use of a naming convention (for the unique name) if you decide to create multiple menus.

Example Naming Convention: myScrollMenu1, myScrollMenu2, myScrollMenu3.

Field: Enter Text For Lines

This is the content that will appear inside the scroll menu(s) once they've been created.

To use this you must type out the content you'd like on a single page and click the button below titled: "Add Line(s)".

Button: Add Line(s)

This will take the content inside the Enter Text For Lines field, and cache it until you create the scroll menu.

Every time you click the button it will erase all content in the Enter Text For Lines field. This is intentional to allow you to type the next page without having to manualy delete the prior.

Button: Create

Hence the name, this button takes all the cached lines you gave it and the unique name and verify everything is valid to be used.

If everything is valid, the data is parsed from the fields and handled eventually creating a scroll menu.

The scroll menu is programatically accessible via: configBot.tags.menuPortal = "<Unique Name You Typed>";

All common errors that you may find will be toasted to you with mostly self explanitory fix instructions.

> For Developers

Everything is set up to be Dimension based.

This "bootstrapper" is setup within a few dimensions.

The Dimension's: home, scrollMenuCreationInput, and scrollMenuCreationGhost.

You can view the code for this bundle via the sheet or system portal inside the chosen Dimension.
